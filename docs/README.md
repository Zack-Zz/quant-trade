## Documentation Areas

- `architecture/`: target architecture, module designs, roadmap, and ultimate-design review.
- `standards/`: project rules, authoring guidance, and governance checklists.
- Future operations runbooks, deployment notes, incidents, and security notes should use `docs/operations/`.
- Future stable facts such as glossary, schemas, ADRs, and diagrams should use `docs/references/`.

Existing root-level documents remain valid. New material should prefer the area layout above.
